Setting up DocFetcher in Eclipse

Follow these instructions to set up the source distribution of DocFetcher in Eclipse:
  • Download the source code for DocFetcher if you haven't already.
  • Download Eclipse 3.4 or newer for your platform. Make sure you download the "Eclipse Classic" edition.
  • Download the version of AJDT (AspectJ Development Tools) that matches your version of Eclipse. For Eclipse 3.4 this would be AJDT 1.6.x.
  • Install AJDT by extracting the package contents into the Eclipse folder. See here for more information. Then run Eclipse with this command: "eclipse -clean". This will make Eclipse load the AJDT plugin.
  • In the Eclipse menu, choose File > Import > Existing Projects into Workspace and select the DocFetcher source code package. Now a new project will be created.
  • Somewhere in the Eclipse preferences you can set the JDK compliance level to 1.5.0. With a 1.6.0 compliance level Eclipse may refuse to compile the program.
  • Set up the referenced libraries. Make sure the correct SWT library is on the classpath (there's two of them, one for Windows and one for Linux).
  • Create a new Run Configuration: Run > Run Configurations.... The working directory of this configuration has to be set to DocFetcher/bin (instead of DocFetcher), because the application will look for the icons folder, which is located inside DocFetcher/bin. Now try to run DocFetcher.
  • If you're able to compile, but can't run and instead get an error message like "Exception in main", without a stacktrace, disable the custom ExceptionHandler by changing the first line in DocFetcher.main to this:
    ExceptionHandler.setEnabled(false);

Running the build script

The Ant build script requires an additional ant task that isn't shipped with the DocFetcher source, the AspectJ Tools. Download it from here and put the "aspectjtools.jar" file in it into the root directory of the newly created DocFetcher project. Now you should be able to run the build script.

To create the Windows installer, boot into Windows, install NSIS and execute build-win-installer.nsi.
