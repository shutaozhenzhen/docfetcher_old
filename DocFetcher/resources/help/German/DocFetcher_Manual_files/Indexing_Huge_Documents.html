Indizierung großer Dokumente

Wenn die Größe eines Dokumentes eine bestimmte Größe überschreitet (das ist abhängig vom Dateityp, z. B. > 5 MB für HTML-Dateien), kann DocFetcher es nicht indizieren. Stattdessen erhalten Sie eine Warnung "Nicht genug Speicher" und DocFetcher überspringt die Datei. Was sie dagegen tun können:

Index nach Neustart aktualisieren

Nachdem DocFetcher die Indizierung beendet hat, starten Sie es erneut und wählen "Aktualisierung" aus dem Kontextmenü des Index, in dem sich die große Datei befindet. Es besteht eine gewisse Wahrscheinlichkeit, dass direkt nach dem Start noch genug Hauptspeicher zur Verfügung steht, um eine solch große Datei zu indizieren.

Erweitern Sie die Speichermenge, die DocFetcher nutzen darf

DocFetcher läuft in einer sogenannten "Java Virtual Machine", für die es eine Begrenzung hinsichtlich des benutzbaren Hauptspeichers gibt - generell als "maximum heap size" bekannt. Falls das Dokument zu groß ist, wird diese Grenze erreicht und die Indizierung schlägt fehl. Sie können diese Grenze durch Änderung der Startparameter des Programms ändern. Die zu ändernde Datei hängt von der von Ihnen verwendeten DocFetcher-Version ab:

  • Windows: DocFetcher.bat - in dem Programmordner, in den DocFetcher installiert wurde (z. B. "C:\Programme\DocFetcher").
  • Linux: DocFetcher.sh - in /usr/share/docfetcher. Sie benötigen hierfür Schreibrechte.
  • Portable: DocFetcher.bat und DocFetcher.sh - in dem Ordner, in den DocFetcher entpackt wurde.
Alle diese Dateien enden mit einer Zeile, die mit  "java -cp" startet. Fügen Sie "-Xmx256m" zwischen "java" und "-cp" ein. Dies setzt den Wert für die "maximum heap size" auf 256 MB, was für fast alle Fälle ausreichen sollte. Nun sollte die letzte Zeile der geänderten Datei wie folgt aussehen:
java -Xmx256m -cp [gefolgt von kryptischen Programm-Befehlen...]

Dann speichern Sie die Datei einfach und starten DocFetcher damit!

... oder überspringen Sie die Datei einfach

Um die Datei von der Indizierung auszuschließen, können Sie sie aus dem Dokumentverzeichnis verschieben oder einen "Datei überspringen"-Ausdruck schreiben der dem Dateinamen entspricht.
